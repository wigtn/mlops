# 일본 이주 + MLOps 커리어 전환 플랜

## 0. 현재 포지션 냉정 평가

| 항목 | 상태 | 시장 평가 |
|---|---|---|
| 나이 | 30세 | 일본 취업 시장에서 여전히 "젊은 축", 전환 충분히 가능 |
| 비자 | 배우자 비자 가능 | **최대 무기**. 취업 제한 없음, 회사 스폰서 불필요 → 지원 가능 풀 3배 이상 |
| 일본어 | N1 기취득(만료), 회화 약함 | 문서/독해 OK, 회의/고객 응대는 부족. 6개월 집중하면 회복 가능 |
| 백엔드 | 초급 | 주니어 포지션은 가능 |
| DevOps | 초급 | K8s/Docker 실무 경험 쌓으면 MLOps 디딤돌 |
| React | 1년 미만 | 풀스택 포지션 지원 카드 |
| AI/ML | 전무 | **기준 공고(Moreh/Rakuten GPUOD)는 현재 0%, 2~3년 로드맵 필요** |

### 핵심 현실
- **Moreh/Rakuten GPUOD 급은 지금 당장은 불가능.** AI 경력 0 + MLE 2~5년 우대 요건 충족 불가.
- **대신 "일본 이주 먼저 → 일본에서 MLOps 전환" 2단 전략이 압도적으로 유리.** 배우자 비자가 있어서 한국에서 MLOps 쌓고 가는 루트보다 일본에서 쌓는 루트가 훨씬 빠름.
- **현 회사 1년 미만 이직은 허용.** 단, 다음 회사는 최소 2년은 붙어있을 것.

---

## 1. 전체 로드맵 (3단계, 총 24~30개월)

```
[Phase 1: 0~6개월]  한국 잔류, 기반 정비 + 일본 이주 실행
[Phase 2: 6~24개월] 일본 내 풀스택/백엔드 포지션 + MLOps 역량 축적
[Phase 3: 24개월~]  MLOps/LLM 인프라 포지션 전환 (Rakuten GPUOD / PFN급)
```

---

## 2. Phase 1 (0~6개월) — 기반 정비 & 일본 이주 실행

### 목표
현 회사 재직 유지하며 (1) 일본어 회화 복구 (2) 기초 AI/MLOps 소양 (3) 배우자 비자 + 이주 실행

### 2-1. 일본어 (최우선)
- **회화 1일 30분**: 일본 거주 배우자와 매일 통화/영상, 의식적으로 일본어만 사용
- **비즈니스 일본어**: 敬語, 会議用語, IT 용어. [NHK Easy News 일일 1건 음독] + [일본 IT 블로그 (Qiita, Zenn) 주 3건 읽기]
- **면접 대비**: 자기소개 / 경력 설명 / 기술 설명 스크립트 일본어 작성 → 암기 → 말하기 반복
- **N1 재취득은 불필요**. 이력서에 "N1 취득 경험(2XXX년)" 명시로 충분

### 2-2. 기술 (일주일 10~15시간, 현 직장 유지하며)
- **AI 기초 (주 5시간)**: Andrew Ng ML Specialization → HuggingFace NLP Course. 코드를 "돌려보는" 수준 목표
- **DevOps/MLOps 기초 (주 5시간)**: Docker 숙련 → Kubernetes (minikube/kind) → GitHub Actions CI/CD
- **포트폴리오 프로젝트 1개**: 간단한 LLM 앱 (예: RAG 챗봇) + Docker + K8s 배포 + GitHub 공개
  - Rakuten/PFN 같은 회사는 "GitHub에 뭐가 있냐"를 본다. 공개 리포 1~2개면 충분한 출발점

### 2-3. 이주 실행
- **배우자 비자 신청** (在留資格認定証明書) — 배우자가 일본에서 입관국 제출. 보통 1~3개월 소요
- **이주 시점 결정**: 현 회사 1년 채우고 퇴사 → 이주 시나리오가 깔끔함 (이직 시 "최소 1년 근무" 충족)
- **이주 전 자금**: 최소 6개월 생활비 (도쿄 기준 1인 150~200만엔 권장)

## 3. Phase 2 (6~24개월) — 일본 정착 + 풀스택/백엔드 포지션

### 목표
일본 내 정식 풀타임 IT 포지션 확보. MLOps는 아직 아님. **일본 경력 1.5~2년 + 기술 스택 업그레이드**.

### 3-1. 타겟 회사 티어
- **Tier A (목표)**: 영어 공용/외국인 많은 회사 — Rakuten, Mercari, LINE(LY Corp), Woven by Toyota, PayPay, Indeed Japan
  - 풀스택/백엔드 주니어~미드 포지션 지원. 배우자 비자 있으면 서류 통과율 상승
- **Tier B (현실)**: 일본 중견 SI/자사 서비스 — Money Forward, freee, SmartHR, DeNA, Cybozu
  - 일본어 비중 높음. N1 회복되면 승산
- **Tier C (백업)**: Human Resocia, SHIFT 등 외국인 SIer
  - 진입은 쉽지만 성장 제한. 최후 카드

### 3-2. 지원 전략
- **잡보드**: TokyoDev, Japan Dev, LinkedIn, BizReach, Wantedly (일본어), Findy
- **지원 시점**: 이주 **3개월 전부터** 원격 면접 시작. 배우자 비자 소지자는 해외 거주 중에도 면접 잘 봐줌
- **이력서**: 일본어 이력서(職務経歴書) + 영문 Resume 둘 다. Moreh 공고 스타일의 "풀스택 잡부" 경험을 "넓은 스택 대응력"으로 재포장

### 3-3. 입사 후 18개월 액션
- **필수 내재화**: K8s 프로덕션 운영, CI/CD 파이프라인, 모니터링(Prometheus/Grafana), 클라우드(AWS or GCP)
- **사내 이동 탐색**: 현 회사 내에 AI/ML 팀이 있으면 사내 이동이 외부 이직보다 훨씬 빠름 (Rakuten, LINE 등 전부 해당)
- **개인 공부**: 주 5~8시간 — vLLM, SGLang 만져보기. 작은 GPU 인스턴스(RunPod/Vast.ai 시간당 $0.5)로 실습. 블로그/Qiita 발신

## 4. Phase 3 (24개월~) — MLOps / LLM 인프라 전환

### 목표
Rakuten GPUOD, PFN, Sakana AI, ELYZA 급 MLOps/LLM 인프라 포지션으로 이동.

### 4-1. 이 시점 필요한 스택 (현실적 최소치)
- vLLM 또는 SGLang으로 LLM 서빙해 본 경험 (개인 프로젝트라도 OK)
- K8s 위에서 GPU 워크로드 돌린 경험
- 분산 추론/학습 개념 이해 (PagedAttention, Tensor Parallelism 등)
- 일본 내 2년 백엔드/DevOps 경력
- 비즈니스 일본어 + 영어

### 4-2. 전환 경로 (우선순위순)
1. **사내 이동**: Phase 2에서 Rakuten/LINE 등에 들어갔다면 GPUOD/AI팀으로 내부 전환. **가장 현실적**
2. **오픈소스 기여**: vLLM, SGLang 리포에 작은 PR 1~2개. Rakuten GPUOD의 면접 가산점 큼
3. **외부 이직**: Phase 2 경력 + 포트폴리오로 Sakana/Recursive/ELYZA 지원

## 6. 자주 빠지는 함정

1. **"AI 먼저 마스터하고 이주한다"** → 최악. 한국에서 AI 독학해봐야 채용시장 증명이 어렵다. 일본에서 커리어 쌓으며 전환이 10배 빠름
2. **"N1 다시 따야 한다"** → 불필요. 회화 복구가 훨씬 중요
3. **Tier C(Human Resocia 등)부터 지원** → 배우자 비자가 있는데 Tier C를 먼저 가면 손해. Tier A부터 두드려라
4. **1년 미만 퇴사를 두려워함** → 배우자 비자 + 일본 이주라는 명분이 있어서 일본 면접에서 마이너스 아님. 오히려 자연스러움
5. **MLOps 포지션에 지금 지원** → 서류 광탈. Phase 2를 건너뛰지 말 것
